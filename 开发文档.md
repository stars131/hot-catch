# 小红书对标创作助手技术开发文档 v0.1

## 1. 项目目标

本项目要实现一个 ChatGPT 式的小红书对标创作助手。

用户可以通过对话输入：

1. 小红书号。
2. 小红书主页链接。
3. 小红书笔记链接。
4. 自己的人设信息。
5. 想发布的主题。
6. 一段想法。
7. 一篇草稿。

系统完成：

1. 自动识别输入类型。
2. 自动查询小红书账号或笔记公开信息。
3. 保存对标账号。
4. 分析单个或多个对标账号。
5. 结合用户人设生成对标建议。
6. 结合用户主题生成小红书图文内容。
7. 保存历史会话和生成内容。

------

## 2. MVP 技术原则

### 2.1 核心原则

1. 对话页优先，不做复杂后台。
2. 先跑通完整闭环，再优化数据源质量。
3. 小红书查询模块必须抽象成适配层，方便后续替换数据源。
4. 所有外部数据都要记录来源、查询时间、可信度。
5. 查询失败不能中断主流程，必须允许用户手动补充。
6. AI 生成必须结构化，方便保存、复制和二次编辑。
7. 用户数据必须隔离。
8. 服务端调用模型和外部数据源，API Key 不暴露到前端。
9. 第一版保留扩展能力，但不做过度工程化。

## 8. 小红书输入识别设计

## 8.1 输入类型

系统需要识别：

```ts
type ParsedInputType =
  | "xhs_id"
  | "xhs_profile_url"
  | "xhs_note_url"
  | "command"
  | "topic"
  | "idea"
  | "draft"
  | "unknown";
```

------

## 8.2 链接识别规则

常见输入：

```text
https://www.xiaohongshu.com/user/profile/xxx
https://www.xiaohongshu.com/explore/xxx
http://xhslink.com/xxxx
小红书号：xxxx
@昵称
```

识别策略：

1. 包含 `xiaohongshu.com/user/profile`：主页链接。
2. 包含 `xiaohongshu.com/explore`：笔记链接。
3. 包含 `xhslink.com`：短链，需要先解析跳转目标。
4. 以 `/` 开头：命令。
5. 文本较短、无空格、无明显句子结构：可能是小红书号。
6. 长文本且带明显观点：可能是想法或草稿。
7. 以“帮我写”“我想发”“生成一篇”为开头：生成内容意图。

------

## 8.3 本地规则 + AI 意图识别

第一层：规则识别。

第二层：AI 识别。

流程：

```text
用户输入
  |
  v
本地规则识别
  |
  |---- 命中明确类型，直接处理
  |
  |---- 不明确
          |
          v
      DeepSeek 意图识别
```

------

## 8.4 识别函数示例

```ts
export type ParsedInput = {
  inputType:
    | "xhs_id"
    | "xhs_profile_url"
    | "xhs_note_url"
    | "command"
    | "topic"
    | "idea"
    | "draft"
    | "unknown";
  intent:
    | "add_benchmark_account"
    | "analyze_account"
    | "generate_report"
    | "generate_advice"
    | "generate_content"
    | "optimize_content"
    | "configure_persona"
    | "general_chat";
  rawText: string;
  normalizedValue?: string;
  confidence: number;
};

export function parseUserInput(text: string): ParsedInput {
  const trimmed = text.trim();

  if (trimmed.startsWith("/")) {
    return {
      inputType: "command",
      intent: mapCommandToIntent(trimmed),
      rawText: text,
      normalizedValue: trimmed,
      confidence: 1,
    };
  }

  if (/xiaohongshu\.com\/user\/profile/.test(trimmed)) {
    return {
      inputType: "xhs_profile_url",
      intent: "add_benchmark_account",
      rawText: text,
      normalizedValue: trimmed,
      confidence: 0.95,
    };
  }

  if (/xiaohongshu\.com\/explore/.test(trimmed)) {
    return {
      inputType: "xhs_note_url",
      intent: "add_benchmark_account",
      rawText: text,
      normalizedValue: trimmed,
      confidence: 0.95,
    };
  }

  if (/xhslink\.com/.test(trimmed)) {
    return {
      inputType: "xhs_note_url",
      intent: "add_benchmark_account",
      rawText: text,
      normalizedValue: trimmed,
      confidence: 0.8,
    };
  }

  if (/^小红书号[:：]\s*/.test(trimmed)) {
    return {
      inputType: "xhs_id",
      intent: "add_benchmark_account",
      rawText: text,
      normalizedValue: trimmed.replace(/^小红书号[:：]\s*/, ""),
      confidence: 0.9,
    };
  }

  if (trimmed.length <= 40 && !/[。！？\n]/.test(trimmed)) {
    return {
      inputType: "xhs_id",
      intent: "add_benchmark_account",
      rawText: text,
      normalizedValue: trimmed.replace(/^@/, ""),
      confidence: 0.65,
    };
  }

  if (trimmed.length > 200) {
    return {
      inputType: "draft",
      intent: "generate_content",
      rawText: text,
      confidence: 0.75,
    };
  }

  return {
    inputType: "topic",
    intent: "generate_content",
    rawText: text,
    confidence: 0.6,
  };
}
```

## 12. DeepSeek 接入设计

## 12.1 Client 封装

```
lib/ai/deepseek-client.ts
type DeepSeekMessage = {
  role: "system" | "user" | "assistant";
  content: string;
};

export async function callDeepSeek(params: {
  messages: DeepSeekMessage[];
  model?: string;
  temperature?: number;
  stream?: boolean;
}) {
  const baseUrl = process.env.DEEPSEEK_BASE_URL || "https://api.deepseek.com";
  const model = params.model || process.env.DEEPSEEK_MODEL || "deepseek-chat";

  const response = await fetch(`${baseUrl}/chat/completions`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${process.env.DEEPSEEK_API_KEY}`,
    },
    body: JSON.stringify({
      model,
      messages: params.messages,
      temperature: params.temperature ?? 0.7,
      stream: params.stream ?? false,
    }),
  });

  if (!response.ok) {
    const errorText = await response.text();
    throw new Error(`DeepSeek API error: ${response.status} ${errorText}`);
  }

  return response;
}
```

## 16. 限流与安全

## 16.1 限流策略

MVP 建议：

1. 单用户每分钟最多 10 次普通对话。
2. 单用户每小时最多 30 次小红书查询。
3. 单用户每小时最多 50 次 AI 生成。
4. 单次输入最大 12000 字。
5. 单次融合分析默认最多使用 5 个对标账号。

虽然用户可以保存无限对标账号，但单次分析不建议无限，否则成本不可控。

------

## 16.2 外部链接安全

处理用户链接时：

1. 只允许 `xiaohongshu.com` 和 `xhslink.com`。
2. 短链最多跳转 5 次。
3. 请求超时 10 秒。
4. 禁止访问内网 IP。
5. 禁止 file 协议。
6. 禁止非 http/https 协议。
7. 记录原始链接。

------

## 16.3 Markdown 安全

前端渲染 Markdown 时：

1. 禁止直接渲染未清洗 HTML。
2. 对链接加 `rel="noopener noreferrer"`。
3. 外链新窗口打开。
4. 代码块安全转义。
5. 防止 XSS。
