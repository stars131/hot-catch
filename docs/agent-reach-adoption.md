Agent Reach 提供面向本地 Agent 的互联网能力层，重点处理渠道选型、安装、健康检查和多后端路由。星迹运行在云端 Next.js/PostgreSQL/BullMQ 架构中，现有产品边界聚焦小红书图文和抖音视频脚本，因此只吸收能增强当前参考导入与来源可靠性的机制。

## 本批实现

普通网页参考导入现在按以下顺序执行：

1. Firecrawl 凭证完整时先调用 Firecrawl。
2. Firecrawl 未取得正文且 `AGENT_REACH_ENABLED=1` 时，Worker 调用固定的 Agent Reach `web` 渠道检查，再通过其当前 Jina Reader 后端读取页面。
3. Agent Reach 未开启、未安装或读取失败时继续安全基础抓取。
4. 所有后端均未取得正文时进入 `waiting_input`，要求用户提供有权使用的摘要。
5. `Idea.evidence` 保存实际后端和脱敏尝试记录，不保存供应商原始错误响应。
6. 未知的后端偏好不会隐藏已有可用候选；重复后端 ID 会在开发阶段直接报错。

Worker 镜像从固定提交安装 Agent Reach 1.5.0，并使用固定 Python 脚本调用。脚本当前只允许 `web` 渠道，只执行只读 `check`；不会运行 `install`、`configure`、Cookie 提取、MCP 配置或其他上游命令。启用前需要确认公开链接可以发送给 Jina Reader，并在部署环境设置：

```env
AGENT_REACH_ENABLED=1
```

## 明确排除

- 运行时 Shell、CLI 自动安装和系统包修改。
- 复用个人浏览器 Cookie、主账号登录态或本机 Chrome 会话。
- 动态 npm/Python 包、任意 MCP 地址、`eval` 和浏览器端外部脚本。
- 把供应商凭证或完整原始响应交给 Skill。
- 绕过平台风控、登录限制或访问授权。

这些路径与星迹的云服务、多用户隔离、加密凭证和受控 Skill 边界冲突。
